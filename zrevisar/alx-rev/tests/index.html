OpenBiblio Test Suite

WARNING: These tests modify the database. DO NOT run the test suite unless you are willing to lose all your data.

  • Unit tests (not in place yet)
  • Functional tests

    As useful as these functional tests are, they aren't as robust as they should be. It's just as likely that failures may indicate errors in the suite as errors in the system. Also, running the tests many times in succession seems to tickle bugs in both IE and Firefox, so a browser restart may be necessary. Look sharp.

    June 2013
    • Hardcoded path! /openbiblio/database_constants.php points to the target database for the test suite.
    • Most tests require that the Install test is run successfully (English language, original test data).
    • Repeating a single test may fail because some tests change the test data.
    • Make sure the browser does not block pop-ups.
    • The Circulation test pops up a secondary window and print dialog. Close it before the Cataloging test starts.
    • Recent versions of Firefox run the test suite well.
    • Recent versions of Internet Explorer require compatibility mode for the tests (not for normal operation).
    • The note below refers to an older version of Internet Explorer.

    IE Note: The Cataloging test seems to trigger some bad interaction between IE and Selenium. I haven't fully investigated yet, but running the test normally almost always results in some sort of 'permission denied' error from IE. However, running the test in single-step mode succeeds.
